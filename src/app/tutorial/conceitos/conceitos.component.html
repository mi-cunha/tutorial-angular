Principais Conceitos no Angular

Se familiarize com alguns conceitos importantes que iremos abordar com frequência durante este tutorial.

Angular CLI

É a Central de Controle usada para operar o Angular na linha de comando.
Através dela criamos um novo projeto e implementamos diversos recursos como componentes, módulos e serviços.

Comandos:

  • ng new: cria um projeto
  • ng serve: inicia um servidor web para testar a aplicação
  • ng generate: cria os recursos citados acima) (pode ser usado abreviado, apenas g
  • ng build: gera o codigo final para a publicação da aplicação

Facilita muito a criação de um novo componente pois já cria todos os arquivos que o compõem e já o configura no módulo, ou seja, o registra em app.module.ts.

REST

Significa Transferência de Estado Representacional É um modelo arquitetural usado para transitar dados entre a aplicação front-end e o servidor.
Faz a comunicação entre os dois.

Benefícios de utilizar REST:

 TypeScript

TypeScript é um superconjunto de JavaScript.
Acrescenta recursos a linguagem JavaScript, como a tipagem, antecipando erros que só seriam pegos ao executar a aplicação, por exemplo.
Tipos fornecem uma maneira de descrever a forma de um objeto, fornecendo melhor documentação e permitindo que o TypeScript valide se seu código está funcionando corretamente.
Foi criada com o objetivo de elevar o nível de código do JS, que era usado apenas do lado cliente e em códigos pequenos.
O Javascript suporta o paradigma de Programação Orientada a Objetos graças ao Typescript, podendo usar variáveis com tipos definidos, criar classes e muito mais.
O Angular faz toda a transformação de TS em JS para o navegador.

Componentes

É aqui que criamos a lógica por trás de toda aplicação para o funcionamento das views.
São classes, ou seja, tem suas propriedades e métodos, escritas em TypeScript e recebem o decorador @Component.
Este decorador tem propriedades como selector, templateUrl e styleUrl:

  • Selector: como identificamos o componente no HTML raiz (app.component.html).
    A tag deve ser usada dessa forma:
    • <nomedoselector> </nomedoselector>
  • TemplateUrl: o nome do arquivo HTML do componente
  • StyleUrl: nome do arquivo de estilo do componente

No arquivo app.module.ts, as referências aos componentes devem estar em declarations!

Rotas

Uma rota está diretamente relacionada a URL, ou seja, também funciona como um localizador de um componente.
Para usar rotas é possível definí-las das seguintes formas:

  • no root module
  • no feature module
  • no módulo de rotas
 Módulos

No contexto do Angular um módulo é a forma de agrupar componentes, diretivas, pipes e serviços que são relacionados.
Um aplicação Angular possui vários módulos, cada um com um propósito.
Essa modularidade no Angular é chamada de NgModules, cada aplicação é composta por pelo menos uma classe NgModule que é o módulo root da aplicação.
Esse módulo root por padrão é chamado de AppModule.
Para definir um módulo Angular devemos usar o decorator de classe @NgModule, que tem como principais propriedades:

  • imports: array com outros módulos que são necessários por componentes declarados nesse módulo.
  • declarations: recebe um array de componentes, diretivas e pipes que fazem parte do módulo.
  • exports: define o conjunto de declarações(componentes, pipes etc) que ficam disponíveis para outros módulos.
  • providers: declara os serviços, se for o módulo root, os serviços ficam disponíveis para toda a aplicação.
 Serviços

Os services contêm métodos que mantêm os dados ao longo da vida de um aplicativo, ou seja, os dados não são atualizados e estão disponíveis o tempo todo.
O principal objetivo de um serviço é organizar e compartilhar lógica de negócios, modelos ou dados e funções com diferentes componentes de uma aplicação Angular.
Criar um service ajuda a dividir o aplicativo da web em unidades lógicas pequenas e diferentes que podem ser reutilizadas.

Diretivas

As diretivas informam ao Angular para anexar um comportamento especificado a um elemento existente.
São usadas com componentes, principalmente para criar tags personalizadas em uma aplicação Angular.
Essas diretivas podem alterar o layout do DOM adicionando e removendo elementos DOM de visualização.

Diretivas Estruturais

Separam completamente a lógica e dados da aplicação, que é escrita em Typescript, da lógica de exibição, que é declarativa no HTML do componente.

  • ngIf: Similiar a estrutura condicional if. No caso, se ‘true’, o elemento HTML deve ser renderizado para o usuário.
  • ngFor:Similar a estrutura de laço for. Permite percorremos objetos iteráveis como arrays.
  • ngSwitch:A estrutura switch é utilizada para verificarmos diversas condições de uma única vez, e executarmos blocos de código de acordo com o resultado dessas condições.
 Pipes

Os pipes são um recurso muito utilizado no Angular para formatar a exibição de valores corretamente.
Existem uma série de pipe disponíveis para uso, disponibilizados pelo próprio Angular. Como por exemplo:

  • DatePipe para datas transformando datas de formato americano em brasileiro
  • UpperCasePipe para tornar letras maiúsculas
  • LowerCasePipepara tornar letras minúsculas
  • CurrencyPipe para moedas, apresentando seu valor em reais (formato BRL) ou o formato que quiser
 Data Binding

É a sincronização automática de dados entre o model e a view , ou seja vínculo entre os dados da classe e o dado exibido no template.
Ao demarcar tags HTMLs com expressões AngularJS, como por exemplo a interpolação, o framework irá associar um watcher para atualizar a view toda vez que o $scope model alterar.
A função tambem tem um listener, onde irá ser chamado quando o valor de message for alterado.
Exemplos:

  • Vincular um evento de clique a um método da classe do componente
  • Interpolação: Faz a conexão entre o componente.ts e o html. Sua sintaxe é substituida pelo valor do atributo.Consiste em adicionar o nome da variável que se deseja chamar entre chaves duplas dentro do elemento onde deseja que seu valor apareça. Então sempre que o valor do atributo mudar, mudará automaticamente na interface.
 Property Binding

É um tipo de data binding, um dos mais utilizados.
Permite que se defina a propriedade de um elemento da view e atualiza o valor da propriedade no componente e o conecta ao elemnto na view.
Sua sintaxe é: O atributo HTML deve ficar entre colchetes e o atributo da classe utiliza aspas.

Event Binding

Significa associação de eventos.
Usamos event binding para vincular um código da aplicação a um certo evento disparado pelo computador.
Podemos responder a qualquer vento disponível na API do DOM que é quem conecta o que vemos na pagina ao código de cada componente da aplicação.
O event binding diz para o Angular qual o evento queremos vincular e o que pretendemos fazer com ele.
Exemplos:

  • (click): clicar com o mouse em um botão
  • KeyUp: quando soltamos uma tecla do teclado
  • KeyDown: quando a tecla do teclado é pressionada
  • KeyPress: enquanto a tela esta sendo pressionada p evento é emitido
